# Quest 1. now-get-to-work

## Название файла Go
`my_answer.sh`

---

## Условия задания 

| **Instruction (оригинал)**                                                        | **Перевод**                                                                  |
| --------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| *"Something terrible happened"*                                                   | *«Произошло что-то ужасное»*                                                 |
| *clone this repo : github.com/01-edu/the-final-cl-test*                           | *клонируй этот репозиторий: `github.com/01-edu/the-final-cl-test`*           |
| *Submit your solution in the file my_answer.sh that will print it when executed.* | *Отправь решение в файле `my_answer.sh`, который при запуске выведет ответ.* |

Смысл: в репозитории спрятана история «ужасного события», и нужно найти правильный ответ и сделать скрипт my_answer.sh, который просто печатает его в терминал.

---

## Решение

```go
echo "Dartey Henv"
```

---

## Логика и пошаговый разбор кода

### 1. Клонирование репозитория
Ты (или проверяющая система) клонируешь репо:
```go
git clone https://github.com/01-edu/the-final-cl-test
```
Внутри этого репозитория по условиям ищется нужная информация, которая в итоге приводит к ответу — имени Dartey Henv.

### 2. Задача файла my_answer.sh
По условию, проверка не интересуется, как ты нашла ответ.
Нужно только, чтобы при запуске:
```go
bash my_answer.sh
```
в терминал выводился корректный текст.

### 3. Вывод строки
```go
echo "Dartey Henv"
```
echo — базовая команда, которая пишет текст в стандартный вывод (терминал).
В кавычках "Dartey Henv" — точное содержимое ответа.

Тесты задания просто проверяют:
запускается ли скрипт → и совпадает ли вывод с ожидаемой строкой.

---

## Глоссарий

| Термин                 | Пояснение                                                     |
| ---------------------- | ------------------------------------------------------------- |
| **Shell script (.sh)** | Файл с командами оболочки (bash/sh), выполняется в терминале. |
| **echo**               | Команда: вывести текст в стандартный вывод.                   |
| **Репозиторий (repo)** | Проект в GitHub, содержащий файлы/задания/тесты.              |
| **Стандартный вывод**  | То, что ты видишь в терминале при запуске программы.          |


---

## Заключение
1. Задание просит только одно: скрипт, который печатает ответ.
2. Твой my_answer.sh делает именно это с помощью одной команды echo.
3. Вся «магия расследования» происходит до этого — при поиске правильного ответа в репозитории, но финальная проверка смотрит только на вывод.
